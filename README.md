# Inventory Management System

CLI-додаток для роботи з інвентарем, який читає CSV та дозволяє робити запити по товарах і категоріях.

## Використання через Docker

1) Збірка імеджа:

```bash
docker build -t inventory-sys .
```

2) Запуск контейнера та робота в CLI:

```bash
docker run -it --rm inventory-sys data/large_inventory.csv
```

Якщо треба використовувати локальні дані (або згенерувати нові) і підкинути їх у контейнер:

```bash
docker run -it --rm -v "$PWD/data:/app/data" inventory-sys data/large_inventory.csv
```

## Локальний запуск

1) (Опціонально) Згенерувати дані:

```bash
python generate_data.py
```

2) Зібрати проєкт:

```bash
g++ -I include/ -O3 main.cpp src/*.cpp -o inventory_sys
```

3) Запустити:

```bash
./inventory_sys data/large_inventory.csv
```

## Що лежить у проєкті

- `data/` - CSV на 1000000 рядків для імітації бази, а також лог з короткою статистикою по категоріях.
- `src/` - реалізація `.cpp` файлів.
- `include/` - заголовки.
- `generate_data.py` - генератор рандомних даних (категорії, артикули, кількість).
- `Dockerfile` - сценарій збірки імеджа.
- `main.cpp` - основний файл з CLI-логікою.

## Обрана структура даних

**Що обрав: Hybrid of Generic (N-ary) Tree with Hash Map Registries.**

- **Чому дерево?** Категорії ієрархічні (наприклад, Clothes -> T-Shirts -> Nike), тому дерево природно відображає зв’язки parent-child і дозволяє агрегувати значення догори.

- **Чому попередні підрахунки?** Дані парсяться один раз на день, а запитів багато. Під час побудови дерева ми одразу оновлюємо `StockCount` і `ArticulCount`, щоб у запитах працювати з уже готовими значеннями ($O(1)$ на вставку).

- **Чому hash map (`std::unordered_map`)?** Дерево вимагає проходу по глибині, але нам потрібен миттєвий доступ до конкретних вузлів. Тому всі створені ноди додатково реєструються в flat-реєстрах для $O(1)$ amortized доступу за рядковим ключем.

- **Економія пам’яті:** у реєстрах зберігаються `std::shared_ptr` на ноди - дані вузлів не дублюються, але доступ залишається швидким.

Загальна логіка: категорії - це дерево з нодами та вказівниками, а для прямого доступу до конкретних елементів використовується `unordered_map`. Також у коді є наслідування класів для зручності користування моделлю.
